# Contributing

InfraLynx documentation accepts fixes, clarifications, architecture notes, and release-history updates.

## Expectations

- Keep documentation aligned with shipped repository behavior.
- Add or update ADRs when architectural decisions change.
- Prefer concise, operationally useful language over marketing copy.
- Preserve navigation consistency in `mkdocs.yml`.

By contributing, you agree to the contributor terms in [CLA.md](CLA.md).
