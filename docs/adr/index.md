# Architecture Decision Records

Architecture Decision Records capture durable engineering decisions that affect how InfraLynx is built, operated, or extended.

## ADR Requirements

Every ADR must include:

- Title
- Status
- Context
- Decision
- Consequences

## ADR Process

- Write an ADR when a decision changes architecture, interfaces, operational behavior, governance, or long-term constraints.
- Keep ADRs simple and stable.
- Update status when a decision is superseded or retired, but do not rewrite history.

## Current ADR Baseline

Initial ADRs already recorded in this repository establish:

- program foundation
- GitHub operating system bootstrap
- documentation system foundation
