# UX Principles

InfraLynx should feel precise, calm, and operationally trustworthy.

## Core UX Rules

- Dark-first UI
- High contrast readability
- Minimal color usage
- Functional layout

## Documentation Experience

Documentation pages should:

- emphasize scanability and hierarchy
- keep navigation stable and predictable
- avoid visual noise
- support long-form technical reading

## Product Alignment

The documentation system is part of the overall product surface. Design choices in docs should reinforce the same clarity, structure, and enterprise tone expected in the product itself.
