# Release Notes

Release notes communicate version-aware changes across the InfraLynx program.

## Release Communication Rules

- Record user-visible, operator-visible, and contributor-facing changes.
- Highlight breaking changes and required migration steps clearly.
- Link to supporting ADRs, issues, or operational guidance when necessary.

## Current State

InfraLynx is in early program foundation stages. Formal release packages have not begun yet, but the release notes structure is in place so later delivery can remain consistent.
